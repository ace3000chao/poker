# 空当接龙 (FreeCell)

## 1. 游戏概述

| 字段 | 值 |
|------|-----|
| 游戏 ID | `freecell` |
| 显示名称 | 空当接龙 |
| 难度 | ★★★★☆ 中高 |
| 预计时长 | 5-20 分钟 |
| 玩家数 | 1 人 |
| 核心机制 | 纯策略、全明牌、临时空间管理 |

空当接龙是所有单人纸牌中策略性最强的——所有 52 张牌一开始全部正面朝上，不存在随机翻牌的运气成分。几乎所有牌局都可解（微软空当接龙内置 32000 局中仅 1 局无解），胜负完全取决于玩家的决策能力。

### 1.1 游戏内规则展示

| 展示项 | 内容 |
|--------|------|
| **🎯 目标** | 将 52 张全明牌按花色从 A 到 K 收集到 4 个基础牌堆 |
| **🃏 布局** | 8 列牌列区（前 4 列 7 张，后 4 列 6 张）+ 4 个空闲位 + 4 个基础牌堆 |
| **🎮 玩法** | ① 点击选中单张或同花色降序牌组 ② 点击目标位置放置 ③ 双击可送入基础牌堆 |
| **✅ 移动规则** | 目标牌比移动牌大 1 点且颜色交替；空闲位每格存 1 张；可移动牌数 = 2^(空闲数 + 空列数×2) |
| **💡 策略** | 尽量少占空闲位，空列可大幅增加可移动牌数 |
| **🏆 胜利** | 4 个基础牌堆全部从 A 到 K 收齐 |

| 区域 | 说明 |
|------|------|
| **牌列区** (Cascade) | 8 列，前 4 列每列 7 张，后 4 列每列 6 张，全部正面朝上 |
| **空闲位** (FreeCell) | 4 个格子，每个最多放 1 张牌 |
| **基础牌堆** (Foundation) | 4 个，按花色从 A→K 升序收集 |

### 2.3 基本规则

**移动规则：**
- 牌列区顶部（每列最后一张）可移动
- 可移动到：
  1. **另一列的顶部** — 目标牌须比移动牌大 1 点且颜色**不同**（红/黑交替）
  2. **空闲位** — 必须为空
  3. **基础牌堆** — 须从 A 开始同花色升序

**同时移动多张牌：**
得益于空闲位和空列，可移动的牌组数量受公式限制：

```
可移动牌数 = 2^(空闲空格数 + 空列数 × 2)
```

| 空闲位可用 | 空列数 | 可一次移动牌数 |
|-----------|-------|--------------|
| 0 | 0 | 1 |
| 1 | 0 | 2 |
| 2 | 0 | 4 |
| 4 | 0 | 16 |
| 4 | 1 | 32 |

**空闲位规则：**
- 每个空闲位最多放 1 张牌
- 空闲位上的牌可以移到牌列或基础牌堆
- 用满空闲位是低效玩法

**空列规则：**
- 空列可以放任意一张牌
- 空列可大幅增加可移动牌数

### 2.4 胜利条件
将所有 52 张牌按花色升序收集到 4 个基础牌堆。

## 6. 技术方案

### 6.1 状态管理

```typescript
interface GameState {
  cascades: Card[][];      // 8 列牌列
  freecells: (Card|null)[]; // 4 个空闲位
  foundations: Card[][];   // 4 个基础牌堆
  selected: {...} | null;
  moves: number;
}
```

### 6.2 关键算法

| 功能 | 说明 |
|------|------|
| 可移动牌数计算 | `Math.pow(2, freeFreecells + emptyCascades * 2)` |
| 批量移动检测 | 选中牌组长度 ≤ 可移动上限 |
| 自动送 Foundation | 若牌可直接送入且不阻断后续 |
| 排列交替校验 | 颜色交替 + 点数递增 1 |

### 6.3 发牌算法

```
1. 用 seed 初始化 PRNG
2. Fisher-Yates 随机排列 [0..51]
3. 前 4 列每列 7 张，后 4 列每列 6 张，全部正面朝上
```
