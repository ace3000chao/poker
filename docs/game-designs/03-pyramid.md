# 金字塔纸牌 (Pyramid Solitaire)

## 1. 游戏概述

| 字段 | 值 |
|------|-----|
| 游戏 ID | `pyramid` |
| 显示名称 | 金字塔纸牌 |
| 难度 | ★★☆☆☆ 简单 |
| 预计时长 | 2-5 分钟 |
| 玩家数 | 1 人 |
| 核心机制 | 配对抗消、逐层解锁 |

金字塔纸牌是一种快节奏的配对消除类单人纸牌游戏。28 张牌排成一个金字塔，玩家需要将两张点数之和为 13 的牌配对消除。全部消除即为胜利。

### 1.1 游戏内规则展示

| 展示项 | 内容 |
|--------|------|
| **🎯 目标** | 消除金字塔中全部 28 张牌 |
| **🃏 布局** | 7 行金字塔（第 1 行 1 张 → 第 7 行 7 张）+ 手牌堆 24 张 |
| **🎮 玩法** | ① 点击两张未被压住的牌使其点数之和为 13 ② K（13 点）可单独点击消除 ③ 无牌可消时翻手牌堆 |
| **✅ 配对表** | A=1, 2=2, ..., 10=10, J=11, Q=12, K=13；配对组合：1+12, 2+11, 3+10, 4+9, 5+8, 6+7 |
| **💡 提示** | 只有底部无遮挡的牌才能操作；弃牌堆顶牌也可参与配对 |
| **🏆 胜利** | 金字塔 28 张牌全部消除 |

## 2. 规则详情

### 2.1 牌具
- 标准 52 张扑克牌

### 2.2 点数对应关系（和为 13）

| 牌面 | 点数 |
|------|------|
| A | 1 |
| 2-10 | 2-10 |
| J | 11 |
| Q | 12 |
| K | 13（可单独消除）|

- **K（13 点）** 可以单独消除（13 = 13 + 0）
- 其他牌必须两两配对消除（如 6+7, A+Q, 2+J 等）

### 2.3 布局

金字塔结构（7 行，每行递增，共 28 张），下方为手牌堆（24 张）。

```
           ┌──┐
          │ 1 │           ← 第 1 行
         ┌──┐┌──┐
        │ 2 ││ 3 │        ← 第 2 行
       ┌──┐┌──┐┌──┐
      │ 4 ││ 5 ││ 6 │     ← 第 3 行
     ┌──┐┌──┐┌──┐┌──┐
    │ 7 ││ 8 ││ 9 ││10│  ← 第 4 行
    ...（共 7 行 28 张）
```

### 2.4 消除规则

- 只能消除**完全未被覆盖**的牌（上方无其他牌压着）
  - 第 7 行（最底层）全部可操作
  - 上层牌只有在其下方两张牌都被消除后才可操作
- 消除的两张牌点数之和 = **13**；K（13）可单独消除
- 弃牌堆最上面的牌可参与配对消除
- 弃牌堆内部不可互相配对

### 2.5 胜利条件
金字塔中 28 张牌全部消除。

## 6. 技术方案

### 6.1 状态管理

```typescript
interface PyramidState {
  pyramid: Card[][];          // 7 行，行长度 1..7
  pyramidExposed: boolean[][]; // 每张牌是否可操作
  stock: Card[];
  waste: Card[];
  selected: Card | null;
  cleared: number;
}
```

### 6.2 关键算法

| 功能 | 说明 |
|------|------|
| 可操作判定 | 底层全可操作，上层仅当下方两张都消除后解锁 |
| 配对检测 | `point(p1) + point(p2) === 13` |
| K 单消检测 | `point(k) === 13` |
| Game Over 检测 | 所有可操作牌 + 弃牌顶牌无法配对、手牌堆为空 |

### 6.3 覆盖关系

`pyramid[row][col]` 被 `pyramid[row+1][col]` 和 `pyramid[row+1][col+1]` 覆盖。
