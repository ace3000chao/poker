## 三、表结构详细定义

### 3.1 users（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | 用户ID |
| `phone` | VARCHAR(20) | UNIQUE, NOT NULL | 手机号（登录账号） |
| `password_hash` | VARCHAR(128) | NULL | 密码哈希（可选，不使用时为NULL） |
| `nickname` | VARCHAR(50) | NULL | 昵称（可选） |
| `role` | VARCHAR(20) | DEFAULT 'user' | 角色：user / admin |
| `points` | INTEGER | DEFAULT 0 | 当前积分余额 |
| `jwt_token` | TEXT | NULL | JWT Token（可为空） |
| `jwt_expires_at` | DATETIME | NULL | JWT Token过期时间 |
| `jwt_refresh_token` | TEXT | NULL | Refresh Token |
| `jwt_refresh_expires_at` | DATETIME | NULL | Refresh Token过期时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 最后更新时间 |
| `last_login_at` | DATETIME | NULL | 最后登录时间 |

**索引**：
- `idx_phone` ON `users(phone)`

**说明**：
- 同一手机号可多次登录（用验证码）
- `password_hash` 为NULL表示该用户只用验证码登录
- `password_hash` 有值表示同时支持密码登录

---

### 3.2 cards（扑克牌表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | 牌ID |
| `card_key` | VARCHAR(20) | UNIQUE, NOT NULL | 牌键：hearts_A、spades_K 等 |
| `suit` | VARCHAR(10) | NOT NULL | 花色：hearts/spades/clubs/diamonds |
| `rank` | VARCHAR(5) | NOT NULL | 点数：A/2-10/J/Q/K |
| `alumni_name` | VARCHAR(50) | NOT NULL | 校友姓名 |
| `graduation_year` | INTEGER | NULL | 毕业年份 |
| `college` | VARCHAR(100) | NULL | 学院 |
| `major` | VARCHAR(100) | NULL | 专业 |
| `company_name` | VARCHAR(100) | NULL | 公司名称 |
| `position` | VARCHAR(100) | NULL | 职位 |
| `industry` | VARCHAR(50) | NULL | 行业领域 |
| `business_desc` | VARCHAR(200) | NULL | 创业简要描述（50字内） |
| `avatar_url` | VARCHAR(500) | NULL | 头像图片URL |
| `contact_phone` | VARCHAR(20) | NULL | 联系电话 |
| `wechat` | VARCHAR(50) | NULL | 微信号 |
| `email` | VARCHAR(100) | NULL | 邮箱 |
| `company_address` | VARCHAR(200) | NULL | 公司地址 |
| `founded_year` | INTEGER | NULL | 公司创立年份 |
| `team_size` | VARCHAR(20) | NULL | 团队规模 |
| `latest_news` | TEXT | NULL | 最新动态（富文本） |
| `alumni_quote` | VARCHAR(500) | NULL | 个人感言 |
| `extra_data` | TEXT | NULL | 扩展数据（JSON格式，存储可选字段） |
| `is_published` | BOOLEAN | DEFAULT TRUE | 是否公开 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `idx_card_key` ON `cards(card_key)` UNIQUE
- `idx_suit_rank` ON `cards(suit, rank)`
- `idx_industry` ON `cards(industry)`

**说明**：
- `card_key` 格式：`{suit}_{rank}`，如 `hearts_A`、`spades_K`
- `extra_data` 为JSON格式，存储校友的扩展信息（联系信息等）
- 52张普通扑克牌使用本表

---

### 3.3 special_cards（特殊牌表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | ID |
| `type` | VARCHAR(10) | NOT NULL | 类型：king（大王）/ joker（小王） |
| `title` | VARCHAR(100) | NOT NULL | 标题（如：中山职业技术学院） |
| `subtitle` | VARCHAR(100) | NULL | 副标题（如：创新创业学院） |
| `logo_url` | VARCHAR(500) | NULL | Logo图片URL |
| `motto` | VARCHAR(200) | NULL | 校训/院训 |
| `description` | TEXT | NULL | 简介（200字内） |
| `contact_phone` | VARCHAR(20) | NULL | 办公电话 |
| `contact_email` | VARCHAR(100) | NULL | 邮箱 |
| `address` | VARCHAR(200) | NULL | 地址 |
| `website_url` | VARCHAR(200) | NULL | 官方网站 |
| `features` | TEXT | NULL | 特色说明（100字内，JSON格式） |
| `extra_data` | TEXT | NULL | 扩展数据（JSON格式） |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `idx_type` ON `special_cards(type)` UNIQUE

**说明**：
- 大王（king）只有一条记录，代表学校
- 小王（joker）只有一条记录，代表学院
- 初期数据固定，后续可扩展

---

### 3.4 games（游戏配置表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | 游戏ID |
| `game_id` | VARCHAR(50) | UNIQUE, NOT NULL | 游戏标识符（如：poker_quiz） |
| `name` | VARCHAR(100) | NOT NULL | 游戏名称 |
| `version` | VARCHAR(20) | NOT NULL | 版本号 |
| `description` | VARCHAR(200) | NULL | 游戏简介 |
| `icon_url` | VARCHAR(500) | NULL | 图标URL |
| `config_path` | VARCHAR(200) | NOT NULL | 配置文件路径 |
| `is_enabled` | BOOLEAN | DEFAULT FALSE | 是否上架 |
| `is_builtin` | BOOLEAN | DEFAULT FALSE | 是否内置游戏 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `idx_game_id` ON `games(game_id)` UNIQUE
- `idx_is_enabled` ON `games(is_enabled)`

**说明**：
- `config_path` 指向 `backend/games/{game_id}/config.json`
- `is_enabled` 控制游戏是否对用户可见

---

### 3.5 game_scores（游戏成绩记录表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | 记录ID |
| `user_id` | INTEGER | FOREIGN KEY -> users.id | 用户ID |
| `game_id` | INTEGER | FOREIGN KEY -> games.id | 游戏ID |
| `game_key` | VARCHAR(50) | NOT NULL | 游戏标识符 |
| `score` | INTEGER | NOT NULL | 游戏得分 |
| `duration` | INTEGER | NOT NULL | 游戏时长（秒） |
| `cards_used` | TEXT | NULL | 使用的牌（JSON数组） |
| `extra_data` | TEXT | NULL | 扩展数据（JSON格式） |
| `points_earned` | INTEGER | DEFAULT 0 | 本次获得的积分 |
| `played_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 游戏时间 |

**索引**：
- `idx_user_game` ON `game_scores(user_id, game_id)`
- `idx_played_at` ON `game_scores(played_at)`

**说明**：
- 每次游戏结束都记录一条成绩
- `cards_used` 为JSON数组格式：`["hearts_A", "spades_K"]`
- `extra_data` 存储游戏特定的额外数据

---

### 3.6 daily_game_counts（每日游戏次数记录表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | ID |
| `user_id` | INTEGER | FOREIGN KEY -> users.id | 用户ID |
| `game_id` | INTEGER | FOREIGN KEY -> games.id | 游戏ID |
| `game_date` | DATE | NOT NULL | 游戏日期 |
| `play_count` | INTEGER | DEFAULT 0 | 当日游戏次数 |
| `updated_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `idx_user_game_date` ON `daily_game_counts(user_id, game_id, game_date)` UNIQUE

**说明**：
- 每天零点后新插入记录或重置 `play_count`
- 按 `settings.max_daily_games` 检查是否超限

---

### 3.7 sms_codes（短信验证码表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | ID |
| `phone` | VARCHAR(20) | NOT NULL | 手机号 |
| `code` | VARCHAR(10) | NOT NULL | 验证码 |
| `purpose` | VARCHAR(20) | NOT NULL | 用途：login / register / reset_password |
| `is_used` | BOOLEAN | DEFAULT FALSE | 是否已使用 |
| `expires_at` | DATETIME | NOT NULL | 过期时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- `idx_phone_purpose` ON `sms_codes(phone, purpose)`
- `idx_expires_at` ON `sms_codes(expires_at)`

**说明**：
- 验证码5分钟内有效
- 使用后 `is_used` 标记为TRUE
- 过期后自动清理（定时任务）

---

### 3.8 login_attempts（登录尝试记录表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | ID |
| `phone` | VARCHAR(20) | NOT NULL | 手机号 |
| `attempt_type` | VARCHAR(20) | NOT NULL | 尝试类型：code / password |
| `is_success` | BOOLEAN | DEFAULT FALSE | 是否成功 |
| `ip_address` | VARCHAR(50) | NULL | IP地址 |
| `attempted_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 尝试时间 |

**索引**：
- `idx_phone_attempted` ON `login_attempts(phone, attempted_at)`

**说明**：
- 用于登录失败锁定：5次失败后锁定15分钟
- 保留最近24小时记录，定时清理

## 五、数据库迁移

使用Flask-Migrate（Alembic）管理数据库版本。

```bash
# 初始化迁移
flask db init

# 生成迁移脚本
flask db migrate -m "initial schema"

# 执行迁移
flask db upgrade
```

## 六、备份策略

```bash
# 备份脚本（每天凌晨3:00执行）
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/data/poker/backups
mkdir -p $BACKUP_DIR
cp /data/poker/poker.db $BACKUP_DIR/poker_$DATE.db

# 保留最近7天
find $BACKUP_DIR -name "poker_*.db" -mtime +7 -delete
```
