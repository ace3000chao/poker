# "我们的王牌"扑克游戏平台 PRD v2.0

## 项目概述

### 项目背景
中山职业技术学院20周年纪念活动核心项目。以扑克牌形式展示52位创业校友信息，打造集数字资产、档案展示、游戏平台于一体的综合性Web平台。

### 项目目标
1. **数字资产管理**：52张扑克牌数字资产的标准化管理与展示
2. **档案展示**：扑克牌珍藏品形式的校友信息展示
3. **游戏平台**：插件式游戏架构，支持第三方开发接入
4. **公共服务**：统一的用户系统、积分体系、排行榜

### 项目名称
**"我们的王牌"** —— 扑克游戏平台

### 核心价值
- 展示学校创业教育成果，献礼20周年校庆
- 建立可持续运营的校友联系与互动平台
- 打造可复用的扑克牌数字资产库，支持游戏生态持续扩展

## 核心功能模块

### 1. 扑克牌档案展示模块

#### 52张普通扑克牌
- **牌面设计**：每张牌显示校友信息（姓名、公司、职位、行业、创业简要描述）
- **花色分类**：
  - 红桃♥：科技、互联网、AI类校友
  - 黑桃♠：电商、贸易、金融类校友
  - 梅花♣：餐饮、服务、零售类校友
  - 方块♦：制造、实体、农业类校友
- **点数排序**：A-K（按毕业年份或创业成就排序）

#### 特殊牌（大王/小王）
| 特殊牌 | 对应单位 | 内容 |
|--------|---------|------|
| **大王** | 中山职业技术学院 | 校徽、校训、学校简介、联系方式、官网 |
| **小王** | 创新创业学院 | 学院Logo、学院简介、联系方式、学院特色 |

#### 展示形式
- **扑克牌墙**：52张牌可视化展示，支持点击查看详情
- **档案详情页**：点击扑克牌显示校友完整信息
- **搜索功能**：按姓名、公司、行业关键词搜索

### 2. 游戏平台模块

#### 插件式架构
- 游戏作为独立模块开发，通过标准接口接入主项目
- 游戏代码并入主项目代码库（非独立部署）
- 每个游戏独立运行，但共享公共服务（用户系统、积分、排行榜）

#### 游戏管理机制
- 后台独立管理每个游戏的**上架/下架**状态
- 上架后所有注册用户可玩，下架后不可见
- 支持多个游戏同时在线

#### 标准插件接口
游戏开发需遵循以下标准接口规范：

```markdown
## 游戏插件标准接口

### 1. 牌面资源获取
- `GET /api/cards` - 获取所有扑克牌列表
- `GET /api/cards/{id}` - 获取单张扑克牌详情（含图片URL）

### 2. 用户数据
- `GET /api/user/profile` - 获取当前用户信息
- `GET /api/user/cards` - 获取用户收藏/拥有的牌（若有收集机制）

### 3. 积分系统
- `POST /api/game/score` - 上报游戏成绩
- `GET /api/leaderboard` - 获取排行榜数据

### 4. 游戏配置
- 游戏启动时读取配置文件，获取游戏参数
- 游戏结果格式统一，便于积分系统处理
```

### 3. 公共服务模块

#### 用户系统
- **注册**：手机号 + 验证码
- **登录**：手机号 + 验证码
- **验证码频率**：60秒内只能发送1次
- **Token**：JWT Token 验证

#### 积分系统
- 用户玩游戏可获得积分
- 积分用于排行榜排名
- 积分规则由各游戏定义，公共服务统一管理

#### 排行榜
- 全局积分排行榜
- 各游戏独立排行榜（可选）
- 实时更新

## 技术架构设计

### 系统架构
```
前端（React + Vite + Tailwind CSS）
    ↓
Nginx 反向代理
    ↓
后端 API（Python Flask）
    ↓
数据库（SQLite）
```

### 关键接口设计
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/send-code` | POST | 发送验证码 |
| `/api/auth/login` | POST | 手机号登录 |
| `/api/cards` | GET | 获取扑克牌列表（公开） |
| `/api/cards/{id}` | GET | 获取扑克牌详情（公开） |
| `/api/user/profile` | GET | 获取用户信息（需登录） |
| `/api/game/score` | POST | 上报游戏成绩 |
| `/api/leaderboard` | GET | 获取排行榜 |
| `/api/admin/games` | GET/POST | 游戏管理（管理员） |
| `/api/admin/games/{id}/toggle` | POST | 游戏上下架（管理员） |

### 安全考虑
- 验证码频率限制（60秒1次）
- JWT Token 验证
- 防止常见Web攻击（SQL注入、XSS等）
- 敏感信息加密存储

## 补充说明

### 扑克牌分配
- 52张普通牌已提前规划对应关系
- 管理员直接分配花色和点数
- 大王=中山职业技术学院，小王=创新创业学院

### 第三方游戏接入流程
1. 程序员阅读《游戏插件接口规范》
2. 按规范开发游戏模块
3. 代码并入主项目仓库
4. 管理员在后台添加游戏并设置上下架

### 关于"收集机制"
本项目**不做**用户收集机制（集齐特定组合换奖励等）。

用户与扑克牌的关系仅为：**查看/了解**（档案展示功能），游戏仅调用牌面数字资产进行娱乐。
