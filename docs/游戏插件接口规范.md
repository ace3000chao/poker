## 一、目录结构规范

每个游戏插件是一个独立目录，置于主项目 `backend/games/` 下：

```
backend/games/
├── game_id_1/          # 游戏目录（英文ID）
│   ├── __init__.py     # 空文件，标识为Python包
│   ├── config.json     # 游戏配置文件
│   ├── routes.py       # 游戏路由（Flask Blueprint）
│   ├── models.py      # 游戏数据模型（可选）
│   └── static/        # 游戏静态资源（可选）
│       └── icon.png   # 游戏图标（32×32 PNG）
│
└── game_id_2/
    └── ...
```

### 游戏目录命名
- 只能使用英文字母、数字、下划线
- 必须唯一，不能与已有游戏ID重复
- 示例：`poker_quiz`、`card_match`、`sorting_game`

## 二、config.json 规范

每个游戏必须有 `config.json`，格式如下：

```json
{
  "id": "game_id",
  "name": "游戏名称",
  "version": "1.0.0",
  "description": "游戏简介（50字内）",
  "icon": "static/icon.png",
  "enabled": true,
  "settings": {
    "max_daily_games": 10,
    "points_per_win": 10,
    "points_per_loss": 2,
    "time_limit_seconds": 300
  },
  "score_schema": {
    "score": "integer",
    "duration": "integer",
    "timestamp": "string",
    "cards_used": "array"
  }
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 游戏唯一标识符 |
| `name` | string | ✅ | 游戏显示名称 |
| `version` | string | ✅ | 游戏版本号，格式：x.y.z |
| `description` | string | ✅ | 游戏简介，50字以内 |
| `icon` | string | ✅ | 图标路径，相对于游戏目录 |
| `enabled` | boolean | ✅ | 是否启用（后台控制） |
| `settings` | object | ✅ | 游戏参数设置 |
| `score_schema` | object | ✅ | 成绩上报的数据格式定义 |

### settings 字段说明

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `max_daily_games` | integer | 10 | 每个用户每天最多玩几次 |
| `points_per_win` | integer | 10 | 获胜获得的积分 |
| `points_per_loss` | integer | 2 | 失败获得的积分 |
| `time_limit_seconds` | integer | 300 | 游戏时间限制（秒） |

### score_schema 字段说明

定义成绩上报的数据格式，每个字段对应成绩数据的一个键：

| 字段 | 类型 | 说明 |
|------|------|------|
| `score` | integer | **必填**：游戏得分 |
| `duration` | integer | **必填**：游戏时长（秒） |
| `timestamp` | string | **必填**：游玩时间（ISO 8601格式） |
| `cards_used` | array | **可选**：使用的牌的ID列表 |

> ⚠️ `score`、`duration`、`timestamp` 为必填字段，由游戏启动时读取配置确认哪些字段必须上报。

## 三、插件注册流程

### 3.1 新游戏接入步骤

1. 在 `backend/games/` 下创建游戏目录
2. 按本规范编写 `config.json` 和 `routes.py`
3. 在 `backend/app.py` 中注册游戏 Blueprint
4. 刷新游戏列表缓存

### 3.2 注册代码示例

```python
# backend/app.py

from flask import Flask
from games.poker_quiz import game as poker_quiz_bp

def create_app():
    app = Flask(__name__)
    
    # 注册游戏插件 Blueprint
    app.register_blueprint(poker_quiz_bp, url_prefix='/api/games/poker_quiz')
    
    return app
```

### 3.3 游戏路由必须实现

| 路由 | 方法 | 说明 |
|------|------|------|
| `/api/games/{game_id}/info` | GET | 获取游戏基本信息 |
| `/api/games/{game_id}/play` | POST | 开始游戏（记录游戏开始） |
| `/api/games/{game_id}/score` | POST | 上报游戏成绩 |

## 四、API 接口详细定义

### 4.1 获取游戏信息

```
GET /api/games/{game_id}/info
```

**请求头**：
```
Authorization: Bearer {jwt_token}
```

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "poker_quiz",
    "name": "翻牌问答",
    "description": "翻转扑克牌，回答校友信息",
    "icon": "/api/games/poker_quiz/static/icon.png",
    "settings": {
      "max_daily_games": 10,
      "points_per_win": 10,
      "points_per_loss": 2,
      "time_limit_seconds": 300
    }
  }
}
```

**错误码**：
| code | 说明 |
|------|------|
| 0 | 成功 |
| 40401 | 游戏不存在 |
| 40402 | 游戏已下架 |

---

### 4.2 开始游戏

```
POST /api/games/{game_id}/play
```

**请求头**：
```
Authorization: Bearer {jwt_token}
```

**请求体**：无

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "game_session_id": "sess_abc123",
    "started_at": "2026-05-16T13:00:00Z",
    "time_limit": 300
  }
}
```

**错误码**：
| code | 说明 |
|------|------|
| 0 | 成功 |
| 40101 | 未登录 |
| 40401 | 游戏不存在 |
| 40402 | 游戏已下架 |
| 42901 | 今日游戏次数已用完 |

---

### 4.3 上报游戏成绩

```
POST /api/games/{game_id}/score
```

**请求头**：
```
Authorization: Bearer {jwt_token}
Content-Type: application/json
```

**请求体**（按 `score_schema` 定义格式）：
```json
{
  "game_session_id": "sess_abc123",
  "score": 85,
  "duration": 120,
  "timestamp": "2026-05-16T13:02:00Z",
  "cards_used": ["hearts_A", "spades_K", "clubs_Q"]
}
```

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "earned_points": 10,
    "total_points": 150,
    "rank": 42
  }
}
```

**错误码**：
| code | 说明 |
|------|------|
| 0 | 成功 |
| 40001 | 参数格式错误 |
| 40002 | game_session_id 不匹配 |
| 40101 | 未登录 |
| 40401 | 游戏不存在 |
| 40402 | 游戏已下架 |
| 42902 | 成绩上报间隔太短 |

## 五、积分计算规则

### 5.1 积分来源

用户积分来源：
1. **游戏获胜**：按 `settings.points_per_win` 获得积分
2. **游戏失败**：按 `settings.points_per_loss` 获得积分

### 5.2 积分扣除

管理员可手动扣除用户积分（后台功能）。

### 5.3 每日游戏次数限制

按 `settings.max_daily_games` 控制每个用户每天的游戏次数，每天零点重置。

## 六、错误码规范

所有游戏插件接口使用统一的错误码体系：

| 错误码 | 说明 |
|--------|------|
| **0** | 成功 |
| **40001** | 参数格式错误 |
| **40002** | 业务逻辑错误（如 session 不匹配） |
| **40101** | 未登录或 Token 无效 |
| **40102** | Token 已过期 |
| **40401** | 游戏不存在 |
| **40402** | 游戏已下架 |
| **42901** | 今日游戏次数已用完 |
| **42902** | 成绩上报间隔太短（需间隔10秒） |
| **50001** | 服务器内部错误 |
